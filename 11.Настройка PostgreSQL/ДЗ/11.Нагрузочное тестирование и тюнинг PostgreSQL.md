- В силу ограничений ресурсов VM, изменил следующие параметры в большую сторону
  ```sql
  -- shared_buffers = 1GB --оставил как есть, тк по дефолту установилось верное значение 25% от 4GB
  postgres=# ALTER SYSTEM SET effective_cache_size TO '4GB'; -- по идее должно быть 3GB как 75% от 4Gb, но поставил 4 тк есть boost
  ALTER SYSTEM
  postgres=# ALTER SYSTEM SET work_mem TO '100MB'; -- по рекомендациям значение должно быть 1/32 от 4GB при этом work_mem * max_connections (40 соеинений) < 4 GB RAM
  ALTER SYSTEM
  -- maintenance_work_mem = 512MB  -- оставил как есть тк по рекомендациям должно ыть 1/16 от RAM, по дефолту значение установилось больше
  -- effective_cache_size = 2 -- не заметил прироста производительнсоти при разных значениях, прирост на уровне погрешности
  -- checkpoint_completion_target = 0.9 --  не заметил прироста производительнсоти при разных значениях, прирост на уровне погрешности
  postgres=# ALTER SYSTEM SET wal_buffers TO '12MB'; -- по умолчанию было 16MB, установил 12MB, увеличение параметра привело к потере производительности, уменьшение дало прирост на 350 транзакций в секунду, силно уменьшать не стал для узбежания частых выгрузок
  ALTER SYSTEM
  postgres=# ALTER SYSTEM SET synchronous_commit TO 'off'; -- отключил ожидание записи на диск при фиксации транзакций, это повысило производительность и увеличило риски потери данных в случае ошибок
  ALTER SYSTEM
  ```
